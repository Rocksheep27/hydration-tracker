# HydrationTracker

HydrationTracker 是一个个人水分摄入管理小工具，最终目标是做成可以在 iPhone 上使用的日常记录工具。

它可以记录每天来自白水、饮料和食物的水分摄入，并估算每日总水分摄入量。未来需要支持提醒功能，帮助用户更稳定地补水。

## 当前状态

V1 命令行原型、V2 本地 Web App 和 V3.8 历史记录管理均已完成。PWA 现分为“今日 / 历史 / 备份”三个视图，数据继续保存在当前浏览器的 localStorage 中。下一阶段 V3.9 将研究固定 HTTPS 发布方式。

当前包含可在 Mac 终端运行的 V1 命令行版本，以及可以查看今日概览、添加记录和整理今日记录的 V2.3 本地 Web App。项目只使用 Python 标准库，不需要安装第三方依赖，但还不是最终 App。

## 产品方向

本项目已经通过命令行原型验证记录和计算逻辑，并完成 Mac 本地 Web App。后续将适配 iPhone Safari，让用户可以添加到主屏幕，像小程序一样使用。

暂时不做 iOS 原生 App，不使用 Swift，也不做 App Store 配置。

## V1 当前功能

- 在 Mac 终端运行。
- 记录白水、饮品和食物。
- 使用常用食物和饮品库快速输入。
- 使用自然单位及默认估算重量或容量。
- 根据含水率估算水分摄入。
- 查看今日总量、目标完成率和剩余量。
- 查看今日记录明细。
- 删除今日单条记录。
- 清空今日记录，同时保留其他日期数据。
- 将数据保存在本地 JSON 文件中。

当前每日目标使用程序内置的默认值 `2000 ml`。V1 不提供每日目标设置，后续增强阶段再考虑自定义目标。

## V2 Web App 当前功能

- 查看今日总水分、默认目标完成率、剩余量和记录数量。
- 在网页中添加白水、饮品和食物记录。
- 使用 V1 常用项目库、默认份量和含水率估算。
- 查看今日记录明细，包括时间、类型、餐次、名称、数量、单位、估算水分和备注。
- 按记录 `id` 删除今天的单条记录，并要求二次确认。
- 清空今天的全部记录，并要求二次确认。
- 删除和清空后自动刷新今日概览与明细。
- 数据只保存在本地 JSON，不联网、不上传。

## 运行 V1 原型

在 Mac 上打开“终端”，进入 HydrationTracker 项目根目录，然后运行：

```bash
python3 src/hydration_tracker.py
```

可以使用简单菜单：

- 添加白水记录
- 添加饮料记录
- 添加食物记录
- 查看今日进度
- 查看今日记录明细
- 删除某条记录
- 清空今日记录
- 退出

### 输入方式

- 白水、饮料和食物都可以从编号菜单中选择常用项目。
- 选择常用项目后，只需输入杯、碗、个、根、份或片的数量。
- 每个常用项目会自动使用默认重量或容量及含水率。
- 选择“精确输入”后，仍可手动输入名称和 `g`、`ml` 或 `cup`。
- 食物餐次使用中文菜单：早餐、午餐、晚餐、加餐和无餐次。
- 备注可以留空，直接回车即可跳过。
- 水分、完成率和剩余量最多显示 1 位小数。

### 查看和整理今日记录

- “查看今日记录明细”会显示序号、时间、名称、原始数量和单位、换算后的重量或容量、含水率及估算水分量。
- “删除某条记录”会先显示今天的明细，再按序号选择，并要求输入 `y` 二次确认。其他日期的记录不会被删除。
- “清空今日记录”只清空当天数据，并要求输入 `CLEAR` 二次确认。历史日期数据会保留。
- 查看今日进度后，可以继续查看明细，逐条核对总量。

数据会保存在本地文件：

```text
data/hydration_log.json
```

## 运行 V2.3 本地 Web App

V2.3 提供今日概览、网页添加记录和今日记录整理功能。它只在 Mac 本机运行，数据仅保存在本地 JSON，不联网也不上传。

在 Mac 终端中运行：

```bash
cd /Users/hhxx/Documents/CodexProjects/HydrationTracker
python3 src/web_server.py
```

启动后，在浏览器中打开：

```text
http://127.0.0.1:8000
```

页面显示今日总水分、默认目标完成率、距离目标还差多少及今日记录数量。终端中按 `Control-C` 可以停止服务器。

### 在网页中添加记录

1. 选择白水、饮品或食物。
2. 从常用项目中选择名称。
3. 输入数量，例如 `0.5`、`1` 或 `2`。
4. 选择餐次，备注可以留空。
5. 核对页面显示的估算水分，然后选择“保存记录”。

保存成功后，页面会显示结果，并自动刷新今日总量、完成率、剩余量和记录数量。默认份量、重量、容量和含水率均为日常估算值。

### 查看和整理今日记录

- “今日记录明细”显示时间、类型、餐次、名称、原始数量和单位、估算水分及可选备注。
- 选择某条记录旁的“删除”后，浏览器会要求二次确认；确认后只按该记录的 `id` 删除今天对应记录。
- 选择“清空今日记录”后，浏览器会要求二次确认；确认后只清空今天的数据，其他日期记录会保留。
- 删除或清空成功后，今日概览和记录明细会自动刷新。

V1 命令行版本仍然可用：

```bash
python3 src/hydration_tracker.py
```

V2.3 仍不支持修改已有记录。需要更正记录时，可以删除错误记录后重新添加，或继续使用上述 V1 命令行版本。

## 预览 V3.8 PWA 原型

V3.8 是独立的纯前端原型，不连接 Python 后端，也不读取或修改 V1/V2 数据。页面通过底部导航分为“今日 / 历史 / 备份”三个视图，记录继续保存在当前浏览器的 localStorage 中。

当前 V3 PWA 原型支持：

- 使用 localStorage 在当前浏览器保存和读取记录。
- 默认打开“今日”视图，首页只放今日概览、添加记录和今日明细。
- 显示今日总水分、剩余量、记录数量和目标完成率。
- 使用 CSS 进度环显示完成率；超过 `100%` 时显示满环，文字保留真实百分比。
- 根据完成率显示六档温和鼓励提示，并随记录变化同步更新。
- 添加白水、饮品和食物记录，自动使用默认份量和含水率估算。
- 保存成功后将餐次恢复为默认“无餐次”，已保存记录仍保留原餐次。
- 显示今日记录明细和项目 emoji，例如苹果 🍎、鸡蛋 🥚。
- 二次确认后删除今天的单条记录或清空今天的全部记录。
- 删除和清空不影响其他日期记录。
- “历史”视图使用月历标记有记录日期，并显示当天总水分。
- 月历支持切换上一个月和下一个月，点击日期后显示该日明细。
- 选中今天或过去日期后，可展开“补录这一天”表单，填写类型、项目、数量、餐次、时间和备注。
- 补录记录的 `date` 使用月历选中日期，`time` 使用用户填写的 `HH:MM`；保存后月历、日期明细和近 7 日趋势同步刷新。
- 历史日期明细支持二次确认后删除单条记录，并按日期和记录 `id` 精确匹配。
- 历史日期明细支持“清空这一天”；确认文案会显示选中日期和记录数量，确认后只删除该日期的全部记录。
- “历史”视图使用原生 SVG 显示近 7 日水分折线图和默认 `2000 ml` 目标线；无记录日期按 `0 ml` 显示。
- “备份”视图集中提供 JSON 导出和导入恢复/迁移。
- 适配 iPhone Safari 的窄屏、安全区域、表单输入和触控尺寸。
- 修复“今日明细”标题换行及取消确认后危险按钮颜色残留问题。
- 提供 `manifest.json`、主屏幕图标和基础 Service Worker 静态缓存配置。
- 提供 JSON 导出备份，用于手动备份、换手机、迁移和恢复数据；导出文件名格式为 `hydration-tracker-backup-YYYY-MM-DD.json`。
- 提供 JSON 导入恢复/迁移。导入采用“替换当前本机数据”，不是合并，并在写入前要求二次确认。
- 导入文件解析或校验失败时不会覆盖当前本机数据。
- 支持添加到 iPhone 主屏幕进行临时测试。

三个视图的使用方式：

- “今日”：查看今日概览、进度环和鼓励提示；添加白水、饮品或食物记录；核对今日明细，并可二次确认后删除今日单条记录或清空今日记录。
- “历史”：查看当前月月历并切换月份；查看近 7 日趋势；点击今天或过去日期后，可查看该日明细、补录记录、二次确认后精确删除单条记录，或二次确认后清空该日期全部记录。
- “备份”：导出 JSON 备份，或选择已有 JSON 文件进行替换导入；导入前会二次确认，建议操作前先导出当前数据。

localStorage key 固定为 `hydration_tracker_v3_records`，当前 `schema_version` 为 `1`。
Service Worker 缓存版本与 `schema_version` 分开管理，缓存只用于静态文件，不保存用户记录。导入导出只处理 `hydration_tracker_v3_records` 这一项本地数据。
三个视图共享同一份 records，切换视图不会改变或丢失数据。历史视图已支持月历查看、近 7 日趋势、指定日期明细、指定日期补录、按日期和记录 `id` 删除单条记录，以及按日期清空选中日期的全部记录。清空指定日期需要二次确认，且只影响该日期，不影响其他日期。旧数据清理暂不实现，仅保留为未来可选增强。JSON 导入导出格式保持不变，导出会包含补录记录。

V3.8 三视图页面结构、指定日期补录和清空指定日期已经在真实 iPhone Safari 上手动测试通过。历史视图选中日期后，可以通过“补录这一天”填写时间、类型、常用项目、数量、餐次和可选备注；记录会保存到月历选中的日期。补录成功后，月历、选中日期明细和近 7 日趋势会刷新；补录今天时，今日概览和今日明细也会同步刷新。历史单条删除与“清空这一天”均要求二次确认，取消时不会修改数据。

详细路线见 `docs/roadmap.md`，产品需求见 `docs/product_requirements.md`。


### 在 iPhone 上测试导出和导入

1. 先用上面的临时局域网方式打开页面。
2. 选择“导出备份”，把 JSON 文件保存到“文件”或其他本地位置。
3. 需要恢复或迁移时，选择“选择备份文件”，再点“导入替换”。
4. 导入确认文案会明确提示：当前本机记录会被备份文件替换，建议先导出当前备份。
