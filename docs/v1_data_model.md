# V1 Data Model

## 目标

V1 通过“水分记录”的数据字段和计算规则，验证白水、饮料和食物的水分估算逻辑。

这一阶段使用 Python 命令行界面，不做提醒功能、Web App 或 iOS 原生 App。

## 一条水分记录的数据字段

每一条记录表示一次摄入行为，例如喝了一杯水、喝了一杯牛奶、吃了一份黄瓜。

| 字段 | 类型建议 | 示例 | 说明 |
| --- | --- | --- | --- |
| `id` | string | `rec_001` | 记录编号，保证每条记录可以被识别。 |
| `date` | string | `2026-06-26` | 日期，建议使用 `YYYY-MM-DD` 格式。 |
| `time` | string | `08:30` | 时间，建议使用 24 小时制。 |
| `category` | string | `water` | 摄入类型，可选 `water` / `beverage` / `food`。 |
| `meal` | string | `breakfast` | 餐次，可选 `breakfast` / `lunch` / `dinner` / `snack` / `none`。 |
| `name` | string | `白水` | 名称，例如白水、牛奶、黄瓜、米饭。 |
| `amount` | number | `1` | 用户输入的原始数量。 |
| `unit` | string | `个` | 用户输入的原始单位，例如 `个` / `根` / `碗` / `份` / `片` / `杯` / `ml` / `g`。 |
| `converted_amount` | number | `180` | 换算后用于计算的重量或容量。旧记录可能没有此字段。 |
| `converted_unit` | string | `g` | 换算后的基础单位，通常为 `g` 或 `ml`。旧记录可能没有此字段。 |
| `water_ratio` | number | `0.86` | 日常估算含水率，范围为 0 到 1。 |
| `estimated_water_ml` | number | `154.8` | 估算水分，单位 ml。 |
| `is_estimated` | boolean | `true` | 标记记录是否使用默认份量或含水率估算。旧记录可能没有此字段。 |
| `note` | string | `早餐搭配` | 备注，可为空。 |

## 字段取值规则

### category

- `water`：白水。
- `beverage`：饮料，例如牛奶、咖啡、茶、果汁。
- `food`：食物，例如黄瓜、米饭、鸡蛋。

### meal

- `breakfast`：早饭。
- `lunch`：中饭。
- `dinner`：晚饭。
- `snack`：加餐或其他食物。
- `none`：不属于餐次，常用于白水和饮料。

### unit

- 常用项目保存自然单位：`杯`、`碗`、`个`、`根`、`份`、`片`。
- 精确输入支持 `ml`、`g` 和兼容旧版的 `cup`。
- `converted_unit` 使用 `ml` 或 `g`，供计算逻辑使用。
- 旧版 `cup` 默认按 250 ml 换算。

## 计算规则

### 白水

白水直接按实际毫升数计入。

```text
estimated_water_ml = amount_ml
```

示例：

```text
白水 300 ml = 300 ml 水分
```

### 饮料

饮料默认按容量乘以含水率计算。

```text
estimated_water_ml = amount_ml * water_ratio
```

示例：

```text
牛奶 250 ml，water_ratio = 0.87
estimated_water_ml = 250 * 0.87 = 217.5 ml
```

咖啡、茶、牛奶等饮品可以有不同的 `water_ratio`。V1 中这些比例只用于日常估算，不追求医学级精确。

### 食物

食物按重量乘以含水率计算。

```text
estimated_water_ml = weight_g * water_ratio
```

示例：

```text
黄瓜 100 g，water_ratio = 0.95
estimated_water_ml = 100 * 0.95 = 95 ml
```

这里默认 1 g 水约等于 1 ml 水，因此可以直接把估算结果记为 ml。

### cup 单位转换

如果单位是 `cup`，需要先转换成 ml。

V1 默认：

```text
1 cup = 250 ml
```

示例：

```text
茶 1 cup = 250 ml
茶 water_ratio = 0.99
estimated_water_ml = 250 * 0.99 = 247.5 ml
```

### 常用份量单位转换

从常用项目菜单选择时，先把用户输入的自然份量换算成 `g` 或 `ml`，再计算水分。

```text
苹果 1 个 -> 约 180 g
estimated_water_ml = 180 * 0.86 = 154.8 ml

鸡蛋 2 个 -> 约 100 g
estimated_water_ml = 100 * 0.75 = 75 ml
```

原始数量和单位保存在 `amount`、`unit`，换算结果保存在 `converted_amount`、`converted_unit`。

### 缺少含水率

如果食物没有含水率：

- 不自动猜测。
- 提示用户手动输入 0 到 1 之间的含水率估算值。
- 输入有效含水率后才创建和保存记录。

示例：

```text
name = "未知食物"
amount = 100
unit = "g"
water_ratio = 0.80
estimated_water_ml = 80
```

## 示例数据和手动验证

### 示例记录

以下是精确输入示例。精确输入时，原始数量和换算后数量相同。

| name | amount | unit | converted_amount | converted_unit | water_ratio | estimated_water_ml | is_estimated |
| --- | ---: | --- | ---: | --- | ---: | ---: | --- |
| 白水 | 300 | `ml` | 300 | `ml` | 1.00 | 300 | `false` |
| 牛奶 | 250 | `ml` | 250 | `ml` | 0.87 | 217.5 | `true` |
| 黄瓜 | 100 | `g` | 100 | `g` | 0.95 | 95 | `true` |
| 米饭 | 150 | `g` | 150 | `g` | 0.60 | 90 | `true` |
| 鸡蛋 | 100 | `g` | 100 | `g` | 0.75 | 75 | `true` |

### 手动计算

```text
白水：300 ml * 1.00 = 300 ml
牛奶：250 ml * 0.87 = 217.5 ml
黄瓜：100 g * 0.95 = 95 ml
米饭：150 g * 0.60 = 90 ml
鸡蛋：100 g * 0.75 = 75 ml
```

今日总水分摄入：

```text
300 + 217.5 + 95 + 90 + 75 = 777.5 ml
```

如果每日目标是 2000 ml：

```text
目标完成率 = 777.5 / 2000 = 38.875%
还差 = 2000 - 777.5 = 1222.5 ml
```

界面最多显示 1 位小数：

```text
今日总水分摄入：777.5 ml
目标完成率：38.9%
还差：1222.5 ml
```

## V1 已实现功能

V1 已基本完成以下核心流程：

1. 在 Mac 终端运行。
2. 添加白水、饮品和食物记录。
3. 使用常用项目、自然份量和默认估算换算值。
4. 根据含水率估算水分。
5. 查看今日总量、目标完成率和剩余量。
6. 查看今日记录明细。
7. 删除今日单条记录。
8. 清空今日记录，同时保留其他日期数据。
9. 保存到本地 JSON。

当前每日目标固定使用默认值 `2000 ml`。自定义每日目标留到后续增强阶段考虑。
